# 大灾变：净化协议（Cataclysm: Cleanwater Bomb）

本项目是基于  
[CleverRaven/Cataclysm-DDA](https://github.com/CleverRaven/Cataclysm-DDA)  
`cdda/master` 分支 commit：**221c786e7d**  
的独立分支项目。

---

项目概述：

大灾变：净化协议（Cataclysm: Cleanwater Bomb）是一个基于 CDDA 实验版的分支项目。

该项目的宗旨是在保留原作高复杂度生存系统与开放式模拟结构的前提下，本围绕以下方向进行系统性调整：

- 现实逻辑一致性增强  
- 机制可解释性重构  
- 生存体验合理化优化  
- 保持 CDDA 原有系统深度与压迫感  

本项目并不是以简化玩法为目标，而是在原有的机制上，建立更加符合现实，同时对应玩家和怪物双向的，保持游戏性的改动，同时会引入大量现实中的做法和操作逻辑。

---

核心设计原则

1. 现实逻辑一致性优先
所有机制改动需尽可能符合现实世界物理、生理或工程逻辑，并能够被解释。

2. 机制可解释性
任何系统行为必须存在明确原因链，而非隐藏规则或纯游戏性修正。

3. 保留 CDDA 核心复杂度
不削弱系统交互深度，不移除生存压力结构，不降低模拟层级。

4. 玩家体验非对抗性优化
避免因信息不透明或非预期规则导致的无意义失败，但保留合理风险与生存压力。

5. 渐进式系统演化
所有改动必须模块化推进，避免破坏上游结构与长期可维护性。

---

分支边界说明（重要）

为保证长期可维护性，明确以下约束：

- 不以降低难度为目标（改动的结果可以说难度被降低了，但是不能为了降低难度去降低单独，同时也不能为了增加难度而增加难度）  
- 不移除核心生存压力机制  
- 不追求与上游完全兼容  
- 不进行无依据的数值“手感优化”  

核心目标是：

> 在复杂系统中提升机制一致性与可解释性，而非削弱复杂度本身。
